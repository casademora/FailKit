What is FailKit

FailKit is a collection of code and tools you can use to Supercharge your own code and apps. We figured if a little bit of a good thing is good, then a lot must be great!

Features

  • More Fail per line of Code than any other Framework
  • More compiler warnings out of the box than any other Cocoa Framework! (And we've looked!)
  • Every NSObject is a Singleton! Singletons are quite useful, so make all your objects useful by making them Singletons!
  • No more .m files! All the code, class headers AND implementation bodies, are in a single file!

UI Components

Fail UI Components Kit provides enough visual failure that you'll never need to look elsewhere for so much fail!

Fail UI Components Kit Features include:

  • All UI Component delegates are automatically set to your Application Delegate!
  • All Table View delegates forward to a single method tableView:didCallMethod:withOptions: so all your code can be in one single, convenient method!

System Requirements

The FailKit Framework is cross platform. There are no system requirements.

Sample Code

@autorelease
{
    idNSObject * failKit = [FailKit defaultInstance];
    while ([failKit retainCount] > 0)
    {
        [failKit autorelease];
    }
}
